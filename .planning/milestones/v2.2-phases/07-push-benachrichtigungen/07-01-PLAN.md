Aenderungen an NotificationManager.swift in zwei Bereichen:

**A) threadIdentifier fuer Notification-Gruppierung (beide Methoden):**

In `scheduleNotificationsForAccount` und `sendNotificationsForAccount`: Nach dem Erstellen von `UNMutableNotificationContent()`, setze `content.threadIdentifier = "account-\(account.id.uuidString)"`. Dadurch gruppiert iOS die Notifications im Notification Center automatisch nach Account.

**B) Summary-Notification bei 5+ aktivierten Websites pro Account:**

In `scheduleNotificationsForAccount`: Vor der for-Schleife ueber Websites, zaehle aktivierte Websites:
```swift
let enabledWebsites = websites.filter { ws in
    guard let setting = self.notificationSettings[ws.id] else { return false }
    return setting != .disabled
}
let useSummary = enabledWebsites.count >= summaryThreshold
```

Wenn `useSummary == true`:
- Statt Einzel-Notifications: Sammle Stats fuer alle aktivierten Websites in ein Array
- Erstelle EINE Summary-Notification pro Account:
  - `content.title = "\(account.displayName) — Zusammenfassung"`
  - `content.threadIdentifier = "account-\(account.id.uuidString)"`
  - `content.body`: Gesamt-Besucher + Name der Website mit dem hoechsten %-Anstieg (visitors.changePercentage). Format: "42 Besucher gesamt • Top: example.com ↑120%"
  - Falls keine Stats geladen werden konnten: "Tippe um deine Statistiken zu sehen"
  - Notification-ID: `"scheduled-\(account.id.uuidString)-summary"`
  - Trigger: gleicher UNCalendarNotificationTrigger wie Einzel-Notifications (basierend auf dem haeufigsten Setting unter den aktivierten Websites — daily oder weekly)

Wenn `useSummary == false`: Bestehendes Verhalten beibehalten (Einzel-Notifications), aber mit threadIdentifier ergaenzt.

Gleiche Logik auch in `sendNotificationsForAccount` (nonisolated) anwenden — dort UserDefaults-basiert statt @Published Properties. Zaehle enabledWebsites aus dem `settings` Dictionary.

**C) Neue Helper-Methode fuer Account-weite Einstellungen:**

```swift
/// Setzt alle Websites eines Accounts auf das gleiche Setting
func updateAllSettings(for accountWebsiteIds: [String], setting: NotificationSetting) {
    for id in accountWebsiteIds {
        notificationSettings[id] = setting
    }
    saveSettings()
    Task { await scheduleAllNotifications() }
}

/// Prueft ob alle Websites eines Accounts aktiviert sind
func allEnabled(for websiteIds: [String]) -> Bool {
    guard !websiteIds.isEmpty else { return false }
    return websiteIds.allSatisfy { notificationSettings[$0] != nil && notificationSettings[$0] != .disabled }
}
```

Schwelle fuer Summary-Notifications als private Konstante: `private let summaryThreshold = 5`
