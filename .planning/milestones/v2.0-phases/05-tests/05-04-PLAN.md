    **Strategie fuer TTL-Expiry-Test:** Da `CacheWrapper` private ist und `save()` immer `Date() + ttl` als expiresAt setzt, muessen wir eine abgelaufene Cache-Datei manuell auf Disk schreiben. Das JSON-Format ist:
    ```json
    {
      "data": [...],
      "cachedAt": "2026-01-01T00:00:00Z",
      "expiresAt": "2026-01-01T00:01:00Z"  // Vergangenheit
    }
    ```

    **Test 1: testLoadExpiredWebsitesReturnsIsExpiredTrue**
    - Cache-Key fuer websites: `"websites_acc-expired"` (aus `websitesKey(accountId:)` Pattern: `"websites_\(accountId)"`)
    - JSON manuell erstellen mit einem CachedWebsite-Array als `data`, `cachedAt` = Vergangenheit, `expiresAt` = Vergangenheit (z.B. `Date(timeIntervalSince1970: 0)` als ISO8601 String `"1970-01-01T00:00:00Z"`)
    - CachedWebsite JSON-Struktur ermitteln: Aus dem Helper `makeCachedWebsite()` — ein `CachedWebsite` hat `id`, `name`, `domain`, `shareId`, `provider` Felder (von `AnalyticsWebsite` abgeleitet)
    - JSON-String mit diesen Feldern zusammenbauen:
      ```swift
      let expiredJSON = """
      {
        "data": [{"id": "1", "name": "Test Site", "domain": "expired.com", "shareId": null, "provider": "umami"}],
        "cachedAt": "2020-01-01T00:00:00Z",
        "expiresAt": "2020-01-02T00:00:00Z"
      }
      """.data(using: .utf8)!
      ```
    - Datei schreiben nach: `tempDir.appendingPathComponent("websites_acc-expired.json")`
    - `sut.loadWebsites(accountId: "acc-expired")` aufrufen
    - Assertions:
      - `result != nil` (Datei existiert und ist lesbar)
      - `result?.isExpired == true`
      - `result?.data.count == 1`

    **Test 2: testClearExpiredCacheRemovesExpiredEntries**
    - Schritt 1: Frischen (nicht-abgelaufenen) Cache erstellen via `sut.saveWebsites([makeCachedWebsite()], accountId: "acc-fresh")`

      - `sut.loadWebsites(accountId: "acc-old") == nil` (abgelaufener Eintrag geloescht)
