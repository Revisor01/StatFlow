# Requirements: InsightFlow

**Defined:** 2026-03-28
**Core Value:** Nutzer können ihre Website-Analytics sicher und übersichtlich vom iPhone aus überwachen

## v2.3 Requirements

Neue UI-Screens für die in v2.2 implementierten API-Endpoints, Analytics auf echten Websites einrichten, und In-App Erklärungen.

### Data Screens (Umami)

- [ ] **SCREEN-01**: Events-View — Nutzer sehen Event-Liste, Event-Details und Event-Statistiken für Umami-Websites
- [ ] **SCREEN-02**: Reports-View — Nutzer können Funnel-Reports, UTM-Auswertungen, Goal-Tracking und Attribution-Reports einsehen

### Data Screens (Allgemein)

- [ ] **SCREEN-03**: Entry/Exit Pages und erweiterte Session-Details — Nutzer sehen woher Besucher kommen und wohin sie gehen
- [ ] **SCREEN-04**: Plausible Goals + Filter — Nutzer können Goals einsehen und Plausible-Daten nach Dimensionen filtern

### Analytics Setup

- [ ] **SETUP-01**: Analytics-Tracking auf eigenen Websites einrichten — Umami/Plausible Tracking-Code auf mindestens einer Website aktiv, Goals definiert, echte Daten fließen

### In-App Guide

- [ ] **GUIDE-01**: In-App Erklärungen — Nutzer verstehen was Bounce Rate, Referrer, UTM-Parameter, Funnels etc. bedeuten und was sie damit anfangen können

## Out of Scope

| Feature | Reason |
|---------|--------|
| Neue Report-Typen erstellen (Revenue, Performance/CWV) | Selten genutzt auf kleinen Websites |
| Admin-Funktionen (User/Team-Management) | Read-only Analytics-Viewer |
| Plausible Custom Properties | Braucht serverseitiges Setup, zu komplex für v2.3 |
