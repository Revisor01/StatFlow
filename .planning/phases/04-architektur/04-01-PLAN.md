PlausibleAPI.swift komplett umstellen ‚Äî exakt dem UmamiAPI-Pattern folgend:

1. **Klassen-Deklaration aendern** (Zeile 6-7):
   - ENTFERNEN: `@MainActor` Annotation
   - ENTFERNEN: `ObservableObject` Konformanz (nur `AnalyticsProvider` behalten)
   - AENDERN: `class` zu `actor`
   - VON: `@MainActor\nclass PlausibleAPI: ObservableObject, AnalyticsProvider {`
   - ZU: `actor PlausibleAPI: AnalyticsProvider {`

2. **Alle `objectWillChange.send()` Aufrufe entfernen:**
   - In `reconfigureFromKeychain()` (ca. Zeile 98-105): `objectWillChange.send()` loeschen
   - `PlausibleSitesManager.shared.objectWillChange.send()` aus reconfigureFromKeychain ENTFERNEN ‚Äî AccountManager uebernimmt das (siehe Task 2)
   - reconfigureFromKeychain() soll danach ein leerer Funktionskoerper sein (oder nur ein Kommentar "PlausibleAPI reads from Keychain ‚Äî no actor state to update")

3. **Protocol-Properties als nonisolated markieren:**
   - `nonisolated var serverURL: String { ... }` ‚Äî liest aus Keychain, ist threadsafe
   - `nonisolated var isAuthenticated: Bool { ... }` ‚Äî liest aus Keychain
   - `nonisolated var apiKey: String? { ... }` ‚Äî liest aus Keychain
   - `nonisolated let providerType: AnalyticsProviderType = .plausible` ‚Äî Konstante

4. **`private let decoder` ist okay** ‚Äî `let` Konstanten sind threadsafe im actor-Kontext, keine Aenderung noetig.

5. **`private init()` bleibt** ‚Äî static let shared = PlausibleAPI() funktioniert bei actors genauso.

6. **NICHT aendern:** Alle async func Methoden (getAnalyticsStats, getPages, etc.) bleiben unveraendert ‚Äî sie sind jetzt automatisch actor-isoliert statt @MainActor-isoliert.

7. **PlausibleSitesManager-Klasse am Ende der Datei NICHT aendern** ‚Äî die bleibt @MainActor class ObservableObject.

WARNUNG: Stelle sicher, dass `import WidgetKit` oben bleibt (wird in der Datei benoetigt).
