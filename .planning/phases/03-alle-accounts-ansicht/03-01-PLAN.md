Output: Erweitertes Dashboard mit "Alle"-Modus, Provider-Badges auf Cards, und Auto-Account-Switch bei Website-Tap.

private var accountSwitcherMenu: some View {

            ForEach(accounts) { account in
                Label(account.displayName, systemImage: account.icon).tag(account.id)

        }
        Divider()
        Button { showingAddAccount = true } label: {

        let isUmami = accountManager.activeAccount?.providerType == .umami
        Image(systemName: iconName)
            .foregroundStyle(isUmami ? Color.orange : Color.blue)
    }
}

    - Neue Methode `func loadAllAccountsData(dateRange: DateRange, accounts: [AnalyticsAccount]) async -> [String: AnalyticsAccount]`
    - Iteriert ueber alle `accounts` aus `AccountManager.shared.accounts`

    - WICHTIG: Am Ende den urspruenglichen aktiven Account wiederherstellen falls er sich geaendert hat

    - Die Picker-basierte Auswahl muss umgebaut werden, da "Alle" kein UUID-Account ist
    - Statt Picker: Manuelle Buttons im Menu verwenden mit eigenem Checkmark-Handling
    - Erster Eintrag: `Button` mit Label("Alle", systemImage: "rectangle.stack")` und Checkmark wenn `showAllAccounts == true`
    - Dann `Divider()`
    - Dann fuer jeden Account ein `Button` mit `Label(account.displayName, systemImage: account.icon)` und Checkmark-Image wenn dieser Account aktiv und `showAllAccounts == false`
    - Bei Tap auf "Alle": `showAllAccounts = true`, dann `loadAllAccountsData()` aufrufen
    - Bei Tap auf einzelnen Account: `showAllAccounts = false`, `accountManager.setActiveAccount(account)`, `viewModel.loadData(dateRange:)`

    **4. Menu Label (per User-Entscheidung "Alle-Icon wenn Alle aktiv"):**
    - Wenn `showAllAccounts == true`: `Image(systemName: "rectangle.stack").foregroundStyle(.purple)` (oder .primary — Claude's Discretion fuer Farbe)

    **5. Account-Switcher Sichtbarkeit:**
