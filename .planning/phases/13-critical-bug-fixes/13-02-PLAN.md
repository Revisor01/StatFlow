    **Fix 1: updateAccountSites soll NICHT reloadAllTimelines aufrufen**

    In `updateAccountSites` (Zeile 136-162), den Aufruf von `updateWidgetCredentials(for: updated)` (Zeile 155) ersetzen durch einen Aufruf der NUR die Daten schreibt, ohne `reloadAllTimelines`:

    Neue private Methode `syncWidgetData(for:)` einfuehren die nur schreibt:

    ```swift
    /// Schreibt Widget-Credentials und Account-Daten ohne Timeline-Reload
    private func syncWidgetData(for account: AnalyticsAccount) {
        let token: String
        switch account.providerType {
        case .umami:
            token = account.credentials.token ?? ""
        case .plausible:
            token = account.credentials.apiKey ?? ""
        }

        let sharedProviderType: SharedCredentials.ProviderType = account.providerType == .umami ? .umami : .plausible
        SharedCredentials.save(
            serverURL: account.serverURL,
            token: token,
            providerType: sharedProviderType,
            sites: account.sites
        )

        syncAccountsToWidget()
    }
    ```

    Die bestehende `updateWidgetCredentials(for:)` Methode BEHAELT `reloadAllTimelines()` — sie wird nur noch von `applyAccountCredentials` aufgerufen (nach allen async-Operationen).

    In `updateAccountSites` Zeile 155, aendern:
    VON: `updateWidgetCredentials(for: updated)`
    ZU: `syncWidgetData(for: updated)`

    **Fix 2: Sicherstellen dass in applyAccountCredentials der Widget-Reload NACH dem Notification-Post passiert**

    In `applyAccountCredentials` (Zeile 261-306) die Reihenfolge der letzten zwei Zeilen tauschen:

    VON (Zeile 302-305):
    ```swift
    updateWidgetCredentials(for: account)
    NotificationCenter.default.post(name: .accountDidChange, object: nil)
    ```

    ZU:
    ```swift
    // Erst Views benachrichtigen, dann Widget-Reload (alle Writes sind abgeschlossen)
    NotificationCenter.default.post(name: .accountDidChange, object: nil)
    updateWidgetCredentials(for: account)
    ```
